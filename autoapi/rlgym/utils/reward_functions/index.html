Function to compute the reward for a player. This function is given a player argument, and it is expected that the reward returned by this function will be for that player.

  • state (rlgym.utils.gamestates.GameState) – The current state of the game.

get_final_reward(player, state, previous_action)

Function to compute the reward for a player at the final step of an episode. This will be called only once, when it is determined that the current state is a terminal one. This may be useful for sparse reward signals that only produce a value at the final step of an environment. By default, the regular get_reward is used.

Parameters

  • player (rlgym.utils.gamestates.PlayerData) – Player to compute the reward for.

  • state (rlgym.utils.gamestates.GameState) – The current state of the game.

  • previous_action (numpy.ndarray) – The action taken at the previous environment step.

Returns

A reward for the player provided.

Return type

float
